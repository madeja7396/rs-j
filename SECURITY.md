# Security Policy

## Supported Versions

This project currently supports security fixes on the `main` branch and the latest tagged release.
Older tags may not receive fixes.

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

Use one of the following:

- GitHub Security Advisory (preferred):
  https://github.com/madeja7396/rs-j/security/advisories/new
- If advisory creation is unavailable, open a private channel with maintainers first and avoid
  sharing exploit details publicly.

When reporting, include:

- A clear description of the issue and impact.
- Reproduction steps or proof-of-concept.
- Affected version/tag and environment.
- Suggested mitigation (if known).

## Response Process

- We will acknowledge reports as soon as possible.
- We will validate and triage severity.
- We will prepare a fix and disclose responsibly.
- Public disclosure will occur after a patch is available when possible.
